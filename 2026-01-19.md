# 2. uv 명령어

- 주요 명령어

| **명령어**        | **설명**                                       | **주요 용도**                           |
| ----------------------- | ---------------------------------------------------- | --------------------------------------------- |
| **`uv init`**   | 새로운 Python 프로젝트를 시작합니다.                 | `pyproject.toml` 파일과 기본 구조 생성     |
| **`uv venv`**   | 가상 환경을 생성합니다.                              | 현재 디렉토리에 `.venv` 폴더 생성          |
| **`uv add`**    | 프로젝트에 패키지를 추가합니다.                      | 패키지 설치 및 `pyproject.toml`에 자동 등록 |
| **`uv remove`** | 프로젝트에서 패키지를 삭제합니다.                    | 패키지 언인스톨 및 의존성 명단에서 제거       |
| **`uv tree`**   | 설치된 패키지들의 의존성 관계를 보여줍니다.          | 어떤 패키지가 다른 패키지를 불러오는지 시각화 |
| **`uv run`**    | 프로젝트 환경 내에서 명령어나 스크립트를 실행합니다. | 가상 환경을 활성화하지 않고도 즉시 실행 가능  |

- uv 사용시 생성된 파일
  - pyproject.toml
  - uv.lock
- 가상환경 삭제

```
   # Windows 
   Remove-Item -Path .venv -Recurse -Force
   
   # MacOS
   rm -rf .venv 
```

# 2.  `Code Runner` 설정

- Extension "Code Runner" 설치
- settings.json 에 설정 추가

  - Ctrl((Cmd)) + Shift + P :
  - "settings.json" 입력
  - "Open User Settings(JSON)" 을 선택하세요.
- 추가할 내용

```
{
    "code-runner.executorMap": {
        "python": "uv run \"$fullFileName\""
    },
    "code-runner.runInTerminal": true,
    "code-runner.clearPreviousOutput": true,
    "code-runner.respectShebang": false,
}
```

- 에디터 ">" 버튼 오른쪽에서 "Run Code" 를 선택합니다.

# 3. GEMINI.md 설정

- Gemini CLI 또는 Antigravity가 실행할때 참고해야 한 기본 프롬프트
- GEMINI.md 파일안에 작성한다..    위치는

```
~/.gemini/GEMINI.md
```

- 이 프롬프트는 AI가 프로젝트의 컨텍스트를 파악할 때 **가상환경(.venv) 인식**과 **최신 패키지 매니저(uv) 사용**을 최우선으로 하도록 설계되었습니다.

```
## Python Environment & Tooling Guidelines

Whenever you start a new project or open a directory, you must strictly follow these environment detection and package management rules:

### 1. Virtual Environment Detection (.venv)
- **Check:** Look for a `.venv` directory in the project root.
- **Action:** If `.venv` exists, assume this is the active virtual environment.
- **Context:** Use the libraries and interpreter path associated with this environment for all code analysis and execution suggestions.

### 2. 'uv' Package Manager Enforcement
- **Check:** Look for a `pyproject.toml` file in the project root.
- **Action:** If `pyproject.toml` exists, you must strictly use `uv` for all package management and script execution tasks.
- **Command Translation Rules:**
  - DO NOT use `pip install <package>`. Instead, suggest: `uv add <package>`
  - DO NOT use `pip install -r requirements.txt`. Instead, suggest: `uv pip install -r requirements.txt` (or `uv sync` if applicable).
  - DO NOT suggest running scripts with `python script.py`. Instead, suggest: `uv run script.py`
  - When initializing a project or environment, suggest `uv venv` and `uv init`.

### Summary of Priority
1. **Detect** `.venv` -> Set as active environment.
2. **Detect** `pyproject.toml` -> **Enforce** `uv` commands over standard `pip`/`python` commands.
```

- 설명

```
설정 내용 설명 (참고용)

위 영어 지침은 다음과 같은 논리를 담고 있습니다:

1. **가상환경 감지 (`.venv`)**:
  
    - 프로젝트 루트에 `.venv` 폴더가 보이면, 
      즉시 해당 환경을 활성 환경으로 간주하고 코드를 분석하라고 지시합니다.
      
2. **`uv` 강제 사용 (`pyproject.toml`)**:
  
    - `pyproject.toml` 파일이 감지되면, 
      **일반 `pip`나 `python` 명령어를 쓰지 말라**고 명시했습니다.
      
    - 대신 무조건 `uv` 명령어를 사용하도록 "번역(Translation)" 규칙을 정했습니다.
      
        - `pip install` → `uv add`
          
        - `python main.py` → `uv run main.py`
          
```
